# Accès à distance à MySQL

L'outil **MySQL distant** de cPanel permet d'ouvrir l'accès à vos bases de données depuis l'extérieur.

Sur un compte d'[hébergement web](https://www.o2switch.fr/hebergement-illimite/), par défaut les bases de données ne
sont accessibles que depuis l'hébergement.

L'outil mysql distant permet d'**autoriser** l'accès à vos bases de données depuis certaines **adresses IP distantes**
de votre choix. L'authentification par identifiant/mot de passe reste toujours nécessaire.

## Comment ouvrir l'accès à distance aux bases MySQL ?

Pour ouvrir l'accès à distance dans cPanel, il suffit d'indiquer l'adresse IP de l'ordinateur ou du serveur qui va se
connecter sur la base de données.

Si c'est votre ordinateur, via un logiciel comme mysql workbench alors il faut indiquer l'adresse IP trouvable sur
[monip.org](https://monip.org/) par exemple.

:::info
Il n'est **pas possible** d'utiliser le sigle `%` pour autoriser toutes les IPs à se connecter sur votre base de données
pour des questions de sécurité.

A cet instant, il est également **impossible** de mettre en liste blanche un **nom de domaine** ou une
**plage d'adresse IPs**.

Il est uniquement possible de placer en liste blanche **une adresse IPv4**.
:::

Ensuite, pour la connexion sur votre base de données, il faut indiquer en hôte mysql le nom du serveur sur lequel vous
êtes hébergé.

Le nom du serveur est de la forme `quelquechose.o2switch.net`. Exemple `canard.o2switch.net`.
Ce nom de serveur est communiqué dans le mail `Bienvenue chez o2switch` et il faut utiliser ce nom en adresse de connexion
pour le serveur MySQL.

:::warning
L'accès à distance aux bases de données est **toléré** sur nos hébergements mais il y a certaines **restrictions**.

L'accès aux bases de données à distance doit **rester occasionnel**.

Par exemple, pour l'utilisation d'un logiciel comme mysql workbench qui permet de se connecter une fois de temps en
temps à la base de données pour faire des opérations dessus ne pose pas de soucis.

En revanche, il ne sera pas possible d'avoir les données d'un site hébergé chez un prestataire A et la base de données
hébergée sur votre hébergement. De même il ne sera pas possible d'avoir un accès persistant ou H24 sur une base de
données à distance.
:::
