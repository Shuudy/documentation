# Créer et gérer les sous-domaines

L'outil **sous-domaines** permet d'ajouter et de gérer des sous domaines sur votre espace d'hébergement mutualisé,
c'est-à-dire des domaines de la forme : `quelquechose.votre-site.fr`.

Un sous-domaine peut être utile pour créer des adresses de tests, sans avoir besoin de commander des noms de domaine
supplémentaires. Ou cela peut être utilisé pour découper logiquement votre site internet, par exemple en ayant votre
site principal sur `mon-domaine.com` et un blog sur `blog.mon-domaine.com`.

## Configurer un sous-domaine

Pour ajouter un sous-domaine, rendez-vous sur l'outil **sous-domaines** situé dans la catégorie **domaines** dans cPanel puis renseignez les paramètres suivants :
  * `Sous-domaine` : le sous-domaine que vous souhaitez ajouter, par exemple `blog` dans le cas présent.
  * `Domaine` : sélectionnez le site sur lequel vous voulez créer ce sous-domaine. Si l'adresse du site sur lequel vous souhaitez créer le sous-domaine n’apparaît pas dans la liste déroulante, vous devez d'abord ajouter votre domaine avec l'[outil domaines configurés](/cpanel/domaines/configuration-domaine) de cPanel.
  * `Racine des documents` : le dossier qui va contenir les fichiers de votre sous-domaine. Nous recommandons de laisser la valeur qui est pré-remplie par défaut.

:::info
Pour que le sous-domaine fonctionne correctement, votre nom de domaine (ou au moins le sous-domaine) doit pointer vers
nos serveurs. Pour cela, vous pouvez faire un [changement de DNS](/nom-de-domaine).
:::

## Gérer les sous domaines existants

Vous pouvez gérer les sous-domaines existants, vous pouvez en supprimer, faire une redirection vers un autre site
(avec le lien `gérer`) ou changer le répertoire où pointe le domaine sur votre espace.

## Sous-domaines spéciaux

Vous avez sans doute remarqué qu'il y a des sous-domaines qui sont présents que vous n'avez pas créé et que vous ne
pouvez pas supprimer.

Il s'agit de sous-domaines spéciaux, créés par cPanel pour ces propres besoins internes.

Par exemple, si le domaine technique lié à votre hébergement est `dojo0001.odns.fr` et si vous configurez
le nom de domaine `mon-domaine.com` alors cPanel va également créer un sous-domaine de la forme
`mon-domainecom.dojo0001.odns.fr`. Vous pouvez l'ignorer, ça n'impacte pas le fonctionnement de votre domaine.
