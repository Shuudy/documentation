# Transfert de domaine : comment ça marche ?

Si vous avez déjà des noms de domaine chez un registre et que vous souhaitez les **rapatrier intégralement vers
o2switch**, vous pouvez initier une **démarche de transfert de nom de domaine**. 

Un **transfert complet** vous permet de changer de registre de domaine : o2switch récupère la gestion technique et
administrative du domaine (le domaine vous appartient toujours !).

Lorsqu'un nom de domaine est récupéré en gestion chez o2switch, les serveurs DNS sont automatiquement changés pour
configurer les serveurs DNS d'o2switch. Donc cela permet de faire pointer le domaine vers nos services.

En résumé, voici comment se déroule un transfert de domaine :

<Badge text="1" /> Il faut **commander le domaine** depuis [l'espace client
o2switch](https://clients.o2switch.fr/acheter-nom-de-domaine).

Un transfert est payant car le domaine est renouvelé en même temps. Si vous ne souhaitez pas payer, vous pouvez
regarder [la procédure de changement de DNS](/nom-de-domaine/pointage-domaine) à la place. 

<Badge text="2" /> Ensuite, vous devez **débloquer votre nom de domaine** et **récupérer le code de transfert**.

Les deux opérations s'effectuent chez votre prestataire de domaine actuel. Le code de transfert est aussi appelé
**authcode** ou **epp code**. C'est une sorte de mot de passe qui permet d'authentifier la demande de transfert.

<Badge text="3" /> Il faudra **fournir ce code de transfert** dans votre interface client ou au support technique.

<Badge text="4" /> Le transfert de domaine sera initié.

<Badge text="5" /> Environ 5 à 7 jours plus tard, nous devrions récupérer la gestion du domaine.

Cela peut prendre un peu plus de temps (10 jours ouvrés). Parfois l'opération est refusée, cela peut arriver si 
l'ancien registre refuse l'opération de transfert sortant.

<Badge text="5" /> A la fin du transfert, les DNS sont changés pour ns1.o2switch.net et ns2.o2switch.net.

## Comment transférer un domaine chez o2switch ?

Pour demander le transfert d'un nom de domaine existant, vous avez 2 possibilités :

  * Si c'est **en même temps que la commande d'un [hébergement
    o2switch](https://www.o2switch.fr/hebergement-illimite/)**, il suffit de préciser votre nom de domaine lors de votre
    commande en choisissant l'option "hébergement + nom de domaine". 
  * Si vous avez déjà un hébergement et que vous souhaitez rapatrier un nom de domaine, il vous faut suivre la même
    démarche que pour la création d'un nom de domaine, c'est-à-dire allez dans votre interface client, puis dans
    "commander un compte" et "commander un domaine supplémentaire".

De notre côté, voyant que le domaine n'est pas disponible à l'enregistrement, nous allons vous envoyer un email intitulé
"transfert de domaine" dans lequel nous vous demanderons 2 choses :

  * le déblocage de votre nom de domaine chez votre prestataire actuel
  * le code de transfert de votre nom de domaine

Vous devrez répondre à notre message avec le code de transfert et nous confirmer qu'il est bien débloqué (à défaut, nous
vous recontacterons). 

Un transfert complet de domaine est une opération payante car le nom de domaine est renouvelé pendant 1 an en même
temps.

## Comment débloquer le domaine et récupérer le code de transfert ?

Cela dépend d'un prestataire à l'autre, mais vous avez 2 cas possibles :

  * votre registrar propose directement un outil dans son panel d'administration pour gérer cela. Dans ce cas,
    recherchez une outil s'appelant : "déblocage du nom de domaine", "transfert de domaine", "code de transfert",
    "authcode", "unlock domaine" ou similaire (cf. ci-dessous la démarche pour quelques prestataires connus)
  * votre registrar ne propose pas cela via son interface, dans ce cas il faut le lui demander en leur faisant une
    demande de support

## Comment communiquer le code de transfert à o2switch ?

Le code de transfert de votre nom de domaine peut être communiquer à o2switch de deux manières :

  * **Depuis votre espace client**, dans la liste de vos domaines, votre domaine va être noté en "nécessite votre
    attention", suivi d'un bouton **fournir le code de transfert**.
  * **En contactant le support** par ticket ou par email. A noter que si le transfert échoue une première fois, il devra
    être lancé manuellement avec l'assistance du support pour vérifier que tout se passe bien. 

## Combien de temps prend un transfert de domaine ?

En moyenne, un transfert de domaine prend entre **5 et 7 jours**, ce délai dépend de votre registrar actuel qui doit
confirmer l'opération. Parfois, vous pouvez également recevoir un email sur l'adresse que vous avez spécifiée dans le
contact "registrant" de votre domaine pour confirmation. 

Durant ce temps de transfert, votre site internet et vos emails ne sont pas hors ligne. Les configurations restent les
mêmes, si vous aviez fait un [changement de DNS](/nom-de-domaine/pointage-domaine) votre site pointera toujours vers
votre hébergement chez o2switch, à défaut il se trouve encore chez votre prestataire actuel. 

**Si vous êtes pressé**, vous pouvez faire un **[changement de DNS](/nom-de-domaine/pointage-domaine) avant de faire un
transfert de domaine**. Un changement de DNS prend 24h et permet d'avoir l'ensemble de vos services chez o2switch. 

Dès que nous récupérons le nom de domaine, un changement de DNS est effectué de manière automatique, pour que le domaine
pointe vers votre espace.

## Modalité de transfert

Hormis le code de transfert et le déblocage du nom de domaine, il existe quelques règles pour le transfert de domaine :

  * le domaine doit **exister depuis au moins 60 jours**
  * le domaine doit **toujours être valide** lors du lancement du transfert (impossible de transférer un domaine expiré
    par exemple). Nous vous recommandons de laisser 15 jours de marge, cela permet de relancer un transfert en cas de
    problème (refus par exemple). 

## Utiliser mon domaine sans faire de transfert

Si vous souhaitez conserver votre nom de domaine chez votre prestataire actuel et le faire pointer vers votre espace
d'hébergement, sans faire l'opération de transfert évoquée plus haut, vous pouvez consulter [cette page de
documentation](/nom-de-domaine/pointage-domaine).
